Tech stack on this page

A lot has been written about the advantages of static web pages and keeping the footprint low. I wanted to try to generate a setup which would give me all that.

Target

  • Static page generation so that a page could be* copied/edited directly in a text editor
  • Content availability on the client side even without js**
  • HTML templating because of all-the-reasons-that-make-templating-better-than-copying
  • Blogging functionality by generating static pages from a source with markdown files (a folder of files for now)

Status

  • The setup builds static pages (like this one) on server side with Webpack from .md files and .pug templates.

ToDo

  • Make everything look a little nicer
  • Work on meta tags and all the things SEO
  • Move configuration to separate files
  • Build navigation
  • Clean up and publish source on GitHub

* Even if it's possible to edit files directly in a text editor that is not very practical when talking about a multi-page setup. But having the possibility tells me that the page is future proof.
** I do like my javascript and am not aiming at a page without javascript but a page that doens't crash and burn without it.
